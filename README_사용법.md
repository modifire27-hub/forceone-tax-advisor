# 포스원 세무 자문 시스템 - 핵심 엔진 (1단계)

이번 단계 범위: `tax_advisor_engine.py` 핵심 엔진 + 터미널 테스트만 포함합니다.
(Streamlit UI, 포스원 런처 연동은 다음 단계입니다.)

## 1. 사용 전 준비 (최초 1회)

1. 이 폴더(`TAX_ROOT`) 전체를 원하는 위치(예: `D:\Tax_automation_v2\TAX_ROOT`)에 복사하세요.
2. 명령 프롬프트(cmd)를 열고 해당 폴더로 이동한 뒤 패키지를 설치하세요.

   ```
   cd D:\Tax_automation_v2\TAX_ROOT
   pip install -r requirements.txt
   ```

3. `.env.example` 파일을 복사해서 이름을 `.env` 로 바꾸세요.
4. `.env` 파일을 메모장으로 열어 `GEMINI_API_KEY=` 뒤에 실제 Gemini API 키를 입력하고 저장하세요.

   ```
   GEMINI_API_KEY=AIzaSy...실제 키...
   ```

   - `GEMINI_MODEL`, `OUTPUT_DIR`은 입력하지 않아도 기본값으로 동작합니다.
   - `.env` 파일은 절대 외부(깃허브, 메신저 등)로 공유하지 마세요.

5. (선택) 법제처 Open API를 함께 쓰려면 `.env`의 `LAW_API_OC=` 뒤에 보유하신 OC 인증키를 입력하세요.
   - **비워두면** 법제처 연동 없이 `_knowledge` 데이터만으로 기존처럼 동작합니다.
   - 인증키 미발급 시 https://open.law.go.kr 에서 회원가입 후 신청할 수 있습니다.

## 2. 지식 베이스(_knowledge) 채우기

`_knowledge` 폴더 안 5개 파일은 현재 **빈 템플릿**입니다 (제목과 작성 형식만 있음).
각 `.txt` 파일을 열어 실제 세무질의 회신 사례, 업종별 유의사항(약국 포함), 부가세 처리지침, 최신 예규 등을
형식에 맞춰 채워 넣으세요. `.xlsx` 파일은 계정과목 매핑표이며 예시 행이 1개 들어있습니다.

**파일 내용을 수정/저장하기만 하면, 코드 수정 없이 다음 질의부터 자동으로 반영됩니다.**

## 3. 실행

### 3-1. 웹 화면으로 실행 (권장)

매번 `cd`로 폴더 이동 후 `python ...`을 입력하는 게 번거로우시면, 아래 명령으로 브라우저 기반 화면을 띄울 수 있습니다.

```
cd D:\Tax_Advisory_G
streamlit run streamlit_ui.py
```

실행하면 자동으로 브라우저가 열리고 다음 화면이 나타납니다.

- **새 주제 시작 버튼**: 화면 상단에 있으며, 현재 진행 중인 대화 묶음을 "이전 대화"로 보내고
  화면을 깨끗하게 비웁니다. 완전히 다른 주제로 넘어갈 때 누르세요.
- **질문 입력란**: 세무 질의를 입력하고 "조회" 클릭
  - 같은 묶음 안에서 연달아 질문하면 **AI가 이전 질문/답변을 실제로 참고**하여
    꼬리질문(예: "그럼 의료기기는 어떻게 돼?")에 맥락 있게 답변합니다.
  - "새 주제 시작"을 누른 뒤 입력하면 새로운 묶음으로 시작되어 이전 맥락이 섞이지 않습니다.
- **현재 대화**: 지금 진행 중인 묶음의 질문/답변이 화면에 계속 누적되어 보입니다 (최신이 위)
- **이전 대화 보기**: "새 주제 시작"으로 넘어간 과거 묶음들이 여기 모입니다.
  펼쳐서 다시 볼 수 있고, 삭제하기 전까지는 사라지지 않습니다.
- **저장 옵션 (왼쪽 사이드바)**:
  - 저장 폴더 직접 지정 가능 (기본값은 `.env`의 `OUTPUT_DIR`, 처음 입력 후에는 **같은 세션 동안 마지막 입력값을 기억**합니다.
    단, 브라우저를 새로고침하거나 앱을 재시작하면 다시 `.env` 기본값으로 돌아갑니다.)
  - 저장 형식 선택: `md`(마크다운) / `docx`(Word) / `pdf` 중 원하는 것을 복수 선택
  - 기본은 타임스탬프로 자동 파일명이 붙지만, "파일명 직접 지정"을 체크하면 원하는 이름으로 저장 가능
- **종합 문서 생성**: 세 가지 단위로 가능합니다.
  - 현재 진행 중인 묶음만 종합
  - 이전 대화의 특정 묶음 하나만 종합
  - 현재 + 모든 이전 대화를 합쳐 전체 종합
  - 종합된 결과도 같은 방식(파일명/형식 지정)으로 저장할 수 있습니다.

종료할 때는 명령 프롬프트 창에서 `Ctrl + C`를 누르면 됩니다.

> **PDF 저장 관련 참고**: PDF는 한글 출력을 위해 Windows에 기본 설치된 "맑은 고딕" 폰트를 자동으로 사용합니다.
> 별도 설치가 필요 없으며, 폰트를 찾지 못하는 경우(드문 경우)에는 화면에 안내 메시지가 표시되고
> 나머지 형식(md/docx)은 정상 저장됩니다.

## 4. 검증된 사항 (이번 단계에서 확인 완료)

- 신규 `google-genai` SDK 사용 (구 `google-generativeai`는 지원 종료되어 제외)
- `.env` 파일 누락 / API 키 누락 시 명확한 한글 오류 메시지 출력
- `_knowledge` 폴더가 비어있을 때 명확한 오류 메시지 출력
- `.txt`(UTF-8) + `.xlsx` 혼합 로드 정상 동작
- API 호출 실패 시 지수 백오프 재시도 후, 실패 원인을 그대로 보여주고 종료 (무한 대기 없음)
- **법제처 Open API 연동** (선택 기능)
  - 실제 OC 키로 호출 검증 완료 (검색 `lawSearch.do` → 본문조회 `lawService.do`, 2단계 호출)
  - 응답 XML 구조(한글 태그: `조문단위`, `조문번호`, `조문내용` 등)에 맞춘 파싱 검증 완료
  - `.env`의 `LAW_API_OC`가 비어있으면 법제처 연동을 자동으로 건너뜀 (하위 호환)
  - 법제처 API 호출이 실패해도 전체 답변 생성 흐름은 끊기지 않음 (경고만 출력하고 계속 진행)
  - 질문에 부가가치세법/소득세법 등 법령명이 포함되면 자동으로 해당 법령 조문을 조회하여
    `_knowledge` 데이터의 보조 근거(2순위)로 활용
- **Streamlit 웹 UI** (`streamlit_ui.py`)
  - headless 모드로 정상 기동, 헬스체크 정상 응답 확인
  - 세션 내 질의응답 누적 표시, 개별 저장/삭제 기능
  - "전체 종합" 버튼으로 누적 대화를 하나의 자문 문서로 재구성
- **문서 변환** (`doc_export.py`)
  - 마크다운 → Word(.docx) 변환 검증 완료 (헤딩/불릿/굵게 서식 정확히 반영)
  - 마크다운 → PDF 변환 시 Windows 한글 폰트(맑은 고딕) 자동 탐색
  - 폰트를 찾지 못해도 다른 형식 저장은 계속 진행 (에러 격리)

## 5. 법제처 API 동작 방식

1. 질문 안에 `COMMON_TAX_LAWS` 목록(부가가치세법, 소득세법, 법인세법 등)에 해당하는
   법령명이 있는지 단순 키워드 매칭으로 확인합니다.
2. 매칭되면 법제처 검색 API로 해당 법령의 일련번호(MST)를 찾습니다.
3. MST로 본문 조회 API를 호출하여 조문 전체를 가져옵니다.
4. 가져온 조문은 `_knowledge` 데이터와 함께 AI에게 전달되며,
   `_knowledge` 데이터가 항상 1순위 근거, 법제처 조문이 2순위 근거로 사용됩니다.
5. 법제처 API 호출이 실패(네트워크 오류, 키 오류 등)해도 경고만 출력하고
   `_knowledge` 데이터만으로 답변을 계속 생성합니다.

법령명 키워드 목록은 `tax_advisor_engine.py`의 `COMMON_TAX_LAWS`에서 직접 수정/추가할 수 있습니다.

## 5. 법제처 API 동작 방식 (중요 — 실제 동작 범위를 정확히 이해하고 사용하세요)

**가져오는 것**: 법령 "조문 본문"만 가져옵니다. (예: 부가가치세법 제1조, 제2조 전체 텍스트)

**가져오지 않는 것**:
- 예규, 유권해석, 질의회신 — 법제처의 다른 서비스이며 현재 미구현
- 판례 — 미구현
- 국세법령정보시스템(hometax 등)의 데이터 — 법제처와 별개 시스템이며 연동되어 있지 않음

**작동 조건**: 질문 문장에 `COMMON_TAX_LAWS` 목록(부가가치세법, 소득세법, 법인세법 등)의
법령명이 **그대로 포함**되어 있을 때만 작동하는 단순 키워드 매칭입니다.
"법인 차량 한도가 얼마야?"처럼 법령명이 안 들어간 질문에는 법제처가 전혀 호출되지 않습니다.

**근거가 전혀 없을 때 동작 (v1.2 수정)**: `_knowledge`에도 없고 법제처도 안 탔다면,
AI는 절대로 자기 학습 지식(구체적 숫자, 한도, 조항 번호 등)으로 답을 채우지 않고
"확인 불가합니다. 회계사에게 문의해주세요"로만 답하도록 시스템 지침을 강화했습니다.
(이전 버전은 "3순위: 일반 세법 지식"을 허용하여, 근거 없는 질문에도 그럴듯한 숫자를
만들어내는 환각이 발생할 수 있었습니다. 법인 차량 관련 테스트에서 실제로 확인된 문제이며,
이번 수정으로 차단되었습니다.)

**결론**: 정확한 답변을 위해서는 `_knowledge` 폴더에 실제 검증된 내용을 채우는 것이
가장 중요합니다. 법제처 연동은 법령 원문을 빠르게 대조하고 싶을 때 보조적으로 쓰는 기능이며,
예규/판례까지 포함한 종합적인 자동 검색 시스템이 아닙니다.

## 6. 국세청 법령해석(질의회신/예규) 검색 보강 — 선택 기능, 신중히 사용 (v1.3 추가)

**왜 이 기능이 필요했는지**: 법제처 API는 국세청 법령해석에 대해 "목록"(안건명, 안건번호,
해석일자, 링크)만 제공하고, **본문 조회 API 자체가 존재하지 않습니다** (다른 부처는 대부분
목록+본문이 세트로 있는데, 국세청과 재정경제부만 예외입니다 — 법제처 OPEN API 신청 화면에서
실측 확인됨). 안건명만으로는 결론(과세/비과세 등)을 전혀 알 수 없어 실효성이 없었습니다.

**우회 방법과 그 한계**: 안건번호로 일반 Google 검색을 하면 lbox.kr 같은 법률 검색 사이트에
본문 일부가 인덱싱되어 있는 경우가 있습니다. 이를 활용해 Gemini의 "Google Search 도구
(grounding)" 기능으로 본문을 찾아 참고정보로 보강하는 기능을 추가했습니다.

**검색 방식 (v1.4 — OR 검색 효과로 개선)**: 법제처 검색 API는 OR 연산자가 없는 단순
문자열 매칭이라, 여러 단어를 한 번에 합쳐서 보내면 0건이 나오는 경우가 많습니다
(예: "음식점업 매입세액공제 면세재화"는 0건, 반면 "음식점업" 단독은 29건).
이를 보완하기 위해 AI가 질문에서 핵심 키워드를 최대 3개까지 뽑아 **각각 따로
검색을 호출하고 결과를 합치는 방식**(OR 검색과 동일한 효과)으로 만들었습니다.
일부 키워드가 0건이어도 다른 키워드에서 결과를 찾으면 그대로 활용됩니다.

**반드시 이해해야 할 한계**:
- 이건 **공식 데이터가 아닙니다.** Google 검색이 우연히 찾아낸 내용이며, 정확성이 보장되지 않습니다.
- 국세법령정보시스템(taxlaw.nts.go.kr) 자체는 robots.txt로 자동 접근(크롤링)을 차단하고
  있어, 그 사이트를 직접 코드로 가져오는 것은 불가능하고 시도해서도 안 됩니다.
- 검색해도 본문을 못 찾으면 안건명/번호/링크만 제공되고, "본문 미확인"으로 명시됩니다.
- 검색 호출마다 Gemini API 별도 과금이 발생할 수 있습니다.
- 시스템 지침상 이 정보는 항상 3순위(최하위) 근거로 취급되며, 답변에 사용될 경우
  반드시 "검증 필요(미확인 참고정보)"라는 라벨이 함께 표시됩니다.

**기본값은 꺼짐(false)입니다.** 켜고 싶으시면 `.env`에서:

```
ENABLE_NTS_SEARCH=true
```

켜시면 질문과 관련된 국세청 법령해석을 자동으로 찾아 참고정보로 답변에 포함시킵니다.
다만 위 한계를 인지하시고, 답변에 이 라벨이 붙은 내용은 반드시 링크를 직접 확인해
검증한 뒤 위하고T 등 실무에 반영하시기 바랍니다.

## 7. AI 일반지식 답변 + 확정 저장 (v1.5 추가)

**배경**: 1~3순위 근거가 전혀 없는 질문(예: "접대비 한도초과액")에 대해 무조건
"확인 불가"로만 답하면, 일반적인 세무 상식 수준의 질문에도 전혀 도움이 안 되는
문제가 있었습니다. 그래서 다음과 같이 단계를 하나 더 추가했습니다.

- **4순위 — AI 일반지식**: 1~3순위 근거가 부족할 때만, AI가 사전에 학습한 일반
  세법 지식으로 답변합니다. 이 경우 답변에 **"[AI 일반지식 기반 - 미검증]"** 라벨이
  명확히 표시됩니다. 이 라벨이 붙은 내용은 반드시 회계사가 직접 검증해야 합니다.

**확정 저장 (PIN 보호)**: 답변이 정확하다고 확인되면, 답변 영역의
**"지식베이스에 확정 저장"** 버튼으로 그 내용을 `_knowledge` 폴더에 영구히
추가할 수 있습니다. 이후 같은 주제의 질문에는 1순위(가장 신뢰도 높은) 근거로
답변됩니다 — 즉 쓸수록 점점 더 정확해지는 구조입니다.

- 이 버튼은 **PIN을 입력해야만** 실제로 저장이 진행됩니다. 직원이 검증 없이
  실수로 누르는 것을 막기 위한 장치입니다.
- PIN은 사이드바 "확정 PIN 관리"에서 처음 설정하거나 나중에 변경할 수 있습니다.
- PIN은 평문으로 저장되지 않고 해시값으로만 저장됩니다 (`.confirm_pin.txt` 파일).
  이 파일은 다른 사람과 공유하거나 외부(깃허브 등)에 업로드하지 마세요.
- PIN을 잊어버린 경우, `.confirm_pin.txt` 파일을 삭제하면 다시 처음부터 설정할 수
  있습니다 (단, 이전 PIN을 모르는 상태에서도 가능하므로 파일 자체의 보관에 유의하세요).

## 9. 구글 시트 로깅 — 검색 기록 자동 저장 (v1.6 추가)

모든 질의응답을 구글 스프레드시트에 자동으로 한 행씩 누적 기록합니다. 회계사든 직원이든
누구나 그 시트 링크에 들어가면 과거 검색 기록을 바로 찾아볼 수 있습니다.

### 9-1. 사전 준비 (최초 1회, 약 10분 소요)

1. **구글 시트 만들기**: https://sheets.google.com 에서 새 스프레드시트를 만듭니다.
   이름은 자유롭게(예: "포스원 세무자문 검색기록"). 시트 안 내용은 비워둬도 됩니다
   (시스템이 자동으로 헤더를 만듭니다).

2. **시트 ID 확인**: 만든 시트의 주소창 URL을 보세요.
   ```
   https://docs.google.com/spreadsheets/d/1AbCdEfGhIjKlMnOpQrStUvWxYz/edit
                                          └──────────이 부분이 시트 ID──────────┘
   ```

3. **Google Cloud 프로젝트 및 서비스 계정 생성**:
   - https://console.cloud.google.com 접속 (구글 계정으로 로그인)
   - 새 프로젝트 생성 (이름 자유, 예: "posone-tax-ai")
   - 좌측 메뉴에서 "API 및 서비스" → "라이브러리"로 이동
   - "Google Sheets API" 검색 후 활성화
   - "Google Drive API"도 검색 후 활성화
   - "API 및 서비스" → "사용자 인증 정보"로 이동
   - "사용자 인증 정보 만들기" → "서비스 계정" 선택
   - 이름 입력 후 생성 (역할 설정은 건너뛰어도 됩니다)
   - 생성된 서비스 계정 클릭 → "키" 탭 → "키 추가" → "새 키 만들기" → JSON 선택
   - JSON 파일이 다운로드됩니다 (예: `posone-tax-ai-abc123.json`)

4. **시트를 서비스 계정과 공유**:
   - 다운로드한 JSON 파일을 메모장으로 열어서 `client_email` 값을 복사하세요
     (예: `posone-tax-ai@posone-tax-ai.iam.gserviceaccount.com`)
   - 1번에서 만든 구글 시트로 돌아가서 우측 상단 "공유" 클릭
   - 방금 복사한 이메일을 추가하고 권한을 "편집자"로 설정 후 공유

5. **JSON 키 파일을 프로젝트 폴더로 이동**: 다운로드한 JSON 파일을
   `D:\Tax_Advisory_G` 폴더 안에 넣으세요 (파일명은 자유롭게 바꿔도 됩니다).

### 9-2. .env 설정

```
GOOGLE_SHEET_ID=1AbCdEfGhIjKlMnOpQrStUvWxYz
GOOGLE_CREDENTIALS_PATH=D:\Tax_Advisory_G\posone-tax-ai-abc123.json
```

### 9-3. 동작 확인

```
python sheet_logger.py
```

"로깅 활성화됨" 메시지와 함께 테스트 행이 시트에 추가되면 정상입니다. 구글 시트를 열어서
직접 확인해보세요.

이후 Streamlit에서 질문을 입력하면 자동으로 시트에 한 행씩 기록됩니다 (일시, 질문,
답변 요약, 전체 답변, 근거유형, 사용자구분). 사이드바의 "사용자 구분"에서 본인이
회계사인지 직원인지 선택할 수 있습니다 (별도 로그인 기능은 아니며, 기록 구분용입니다).

> **주의**: JSON 키 파일과 `.env`는 절대 외부(깃허브, 메신저 등)에 공유하지 마세요.
> 이 키가 노출되면 누구나 해당 구글 시트에 접근할 수 있게 됩니다.
